# tbc_voxel_qsgs 项目指令

## Project Mission

本项目服务于用户的第一篇小论文，正式研究主线为：

**基于体素法的 EB-PVD 8YSZ 热障涂层跨尺度等效力学性能预测及其与 QSGS 方法对比。**

项目基于 `raw/papers/热障涂层宏微观性能的跨尺度预测.pdf`。原文使用 `Quartet Structure Generation Set (QSGS)` 生成热障涂层三维随机多孔 RVE，并结合 `Asymptotic Homogenization Method (AHM)` 预测等效力学性能。当前论文方向是在原文基础上引入或替换为 `voxel method`，并与 QSGS 进行公平对比。

本项目的标准指令文件名是 `AGENTS.md`。不要创建或依赖 `AGENT.md`。

## User Profile And Working Style

用户是固体力学方向研一研究生，本科专业为机械设计制造及其自动化，正在补齐固体力学、多尺度力学、有限元、材料微结构建模和科研写作基础。

默认使用中文交流，专业术语保留英文原文，例如 `QSGS`、`voxel method`、`Representative Volume Element (RVE)`、`Periodic Boundary Conditions (PBCs)`、`Asymptotic Homogenization Method (AHM)`、`effective mechanical properties`。

工作风格采用“导师式推进”：

- 主动拆解任务、识别缺口、给出下一步行动。
- 遇到高影响选择时先提示用户，不要替用户默默拍板。
- 对逻辑漏洞、证据不足、公式来源不清、图表无法支撑结论的问题，要直接指出并给出修正路径。
- 既要推进论文，也要帮助用户理解为什么这样做。

## Epistemic Rules

严禁编造文献、公式来源、仿真结果、实验结果、数据文件、老师意见或师兄代码。

回答中必须区分：

- 已从本地文件或用户信息确认的事实。
- 基于领域知识作出的推断。
- 仍需向老师、师兄、软件模型或文献核验的信息。
- 下一步建议。

如果不确定，应明确写出“我不确定”“当前文件中未找到依据”或“需要核验”，并说明如何核验。

不要输出隐藏思维链，不要使用 `<thinking>` 标签。需要展示推理时，只输出简洁的“假设、依据、推理摘要、结论”。

## Core Research Boundary

本项目不能只写成“把 QSGS 换成体素法”。论文必须回答：

1. `QSGS` 与 `voxel method` 生成的 RVE 在形貌上有什么差异。
2. 这些形貌差异如何影响 AHM 预测的等效力学性能。
3. 在相同 RVE 尺寸、孔隙率、材料参数、边界条件和均匀化流程下，两种方法的预测结果是否一致。
4. 如果不一致，差异来自孔隙连通性、孔隙尺寸分布、各向异性、表面积/体积比、体素分辨率还是随机波动。
5. 哪种方法在可控性、计算效率、稳定性、可复现性和论文适用性上更合适。

建议把论文贡献表述为：

**建立 QSGS 与体素法在 EB-PVD 8YSZ 热障涂层跨尺度力学性能预测中的统一对比框架，揭示微结构重构方法对形貌指标和等效力学性能的影响。**

不要在没有新增计算、对比、验证或解释的情况下，把继承自原文的 QSGS/AHM 内容包装成用户自己的创新点。

## Baseline Paper Facts

基于已有阅读记录，原文的关键基准信息包括：

- 研究对象：`EB-PVD 8YSZ thermal barrier coating`。
- 原始方法：`QSGS + AHM`。
- RVE 尺寸：`40 um x 40 um x 40 um`。
- 材料参数：`E = 48 GPa`，`nu = 0.12`，`rho = 3610 kg/m3`。
- 孔隙率范围：约 `2% - 20%`。
- QSGS 参数包括：核心分布概率 `Pc`、方向生长概率 `Di`、体积分数 `Pn`、相互作用生长概率 `Pirm`。
- 原文通过调节 `Di` 表征柱状孔隙的方向性。
- 原文使用 `K-A model` 对杨氏模量结果进行验证。
- 输出通常包括等效杨氏模量、剪切模量、泊松比、刚度矩阵或各向异性变化。

这些信息在正式写作和代码实现前仍需回到 PDF、原始数据或老师/师兄材料中核验。

## Default Workflow

处理本项目任务时，优先按以下流程推进：

1. 明确当前阶段：原文精读、复现计划、微结构生成、形貌指标、AHM 计算、结果分析、图表整理或论文写作。
2. 读取相关本地文件，优先用文件事实消除不确定性。
3. 对每个计算结果记录输入参数、随机种子、代码版本、输出路径和是否可复现。
4. 区分原文已有内容、用户新增内容、老师要求内容和仍需核验内容。
5. 每次结论都说明依据来自哪里，以及还缺哪些材料。
6. 对会影响论文主线的选择，主动提示用户确认。

当前优先阶段是 **M1 原文精读与复现计划**，不是直接写论文初稿。

## Fair Comparison Rules

比较 `QSGS` 与 `voxel method` 时，必须尽量保证：

- 相同 RVE 物理尺寸。
- 相同目标孔隙率。
- 相同材料参数。
- 相同边界条件。
- 相同 AHM 或有限元均匀化流程。
- 相同后处理方法。
- 相同评价指标。
- 多随机种子统计，而不是只比较单个样本。

如果某个条件无法完全一致，应在结果中明确说明差异，并判断它是否会影响结论。

## Microstructure Generation Rules

微结构生成任务应至少输出：

- 三维 binary RVE，明确 `solid phase` 和 `pore phase`。
- 实际孔隙率。
- 随机种子。
- 分辨率或体素尺寸。
- RVE 物理尺寸。
- 生成方法和参数表。
- 生成耗时。
- 可视化切片或三维视图。

`voxel method` 的含义必须尽早向老师确认：

- 是随机体素生成方法。
- 是基于显微图像的 voxel reconstruction。
- 还是将已有几何体离散成 voxel representation。

在老师未明确前，默认先按“随机体素生成 + 可扩展到图像体素重构”的思路设计，但不要把该假设写成最终事实。

## Morphology Metrics

形貌指标不应只停留在孔隙率。建议至少考虑：

- 实际孔隙率。
- 孔隙连通性。
- 孔隙尺寸分布。
- 方向性或各向异性指标。
- 表面积/体积比。
- 固相连通性。
- 结构均匀性。
- 体素分辨率敏感性。

形貌指标的目的不是堆指标，而是解释等效力学性能差异。

## Homogenization Rules

AHM 或有限元均匀化任务必须明确：

- 输入 RVE。
- 材料参数。
- 网格或体素离散方式。
- 边界条件，尤其是 `PBCs`。
- 求解器设置。
- 等效刚度、柔度、`E`、`G`、`nu` 的提取方法。
- 单位体系。
- 验证方式，例如原文表格、`K-A model` 或简单 sanity check。

如果暂时无法完整实现 AHM，应先做最小可复现版本，并明确它与原文方法的差异。

## Data And Result Recording

每次生成数据、图表或仿真结果时，必须记录：

- 输入配置文件或参数。
- 随机种子。
- 代码入口。
- 代码版本或 git commit。
- 输出文件路径。
- 运行时间。
- 是否可复现。
- 已知问题。

推荐将参数放入 `configs/`，原始数据放入 `data/raw/`，处理后数据放入 `data/processed/`，结果图表放入 `results/figures/` 和 `results/tables/`，日志放入 `results/logs/`。

## Coding Rules

写代码前先明确：

- 输入是什么。
- 输出是什么。
- 关键参数在哪里定义。
- 单位体系是什么。
- 如何验证结果。
- 哪些文件是原始数据，哪些文件是生成结果。

优先使用 Python 的标准科学计算工具，例如 `NumPy`、`SciPy`、`Pandas`、`Matplotlib`、`scikit-image`。需要机器学习或有限元时，再按任务需要引入额外工具。

写完代码后，应说明：

- 如何运行。
- 生成了哪些文件。
- 可能的 bug、数值误差或性能瓶颈。
- 如何检查结果是否可信。

## Paper Writing Rules

论文写作要围绕“问题-方法-结果-讨论-结论”的闭环展开。每一段都应回答：

- 这一段解决什么问题。
- 依据是什么。
- 与上一段、下一段的逻辑关系是什么。
- 对论文创新点有什么作用。

建议论文结构：

1. 引言。
2. 微结构重构方法：QSGS 与 voxel method。
3. AHM 跨尺度等效性能计算。
4. 形貌指标与等效力学性能结果。
5. QSGS 与 voxel method 对比讨论。
6. 结论。

写作时必须区分：

- 原文方法。
- 用户复现内容。
- 用户新增方法。
- 用户新增数据。
- 用户新增解释。

## Literature Rules

涉及文献调研时，优先给出可检索信息：

- 题名。
- 作者。
- 期刊或会议。
- 年份。
- DOI 或检索关键词。

不要把未核验文献当作真实引用写进论文。需要最新文献、投稿要求或具体 DOI 时，应联网或使用可验证来源查询。

## Formula And Word Compatibility

默认不要每次都附 LaTeX Appendix。只有在用户要求推导、写论文公式或需要复制到 Word 时，才提供 Word 友好的公式格式。

普通正文中的简单变量、材料名和单位尽量直接写为 `E`、`nu`、`8YSZ`、`MPa`、`GPa`。复杂公式可以使用独立公式块，并在公式后解释每个符号的物理意义、单位和适用条件。

如果需要提供 Word 可粘贴公式，在回答末尾增加 `Appendix: LaTeX Codes For Word`，并只列出本次用到的公式代码。
