# tbc_voxel_qsgs 项目指令

## Project Mission

本项目目标是产出用户的第一篇小论文。当前正式研究主线为：

**基于体素法的 EB-PVD 8YSZ 热障涂层跨尺度等效力学性能预测及其与 QSGS 方法对比。**

项目基于 `热障涂层宏微观性能的跨尺度预测.pdf` 展开。原文使用 `Quartet Structure Generation Set (QSGS)` 生成三维随机孔隙 RVE，并结合 `Asymptotic Homogenization Method (AHM)` 预测等效力学性能。本项目的核心新增工作不是简单“替换 QSGS”，而是系统比较 **voxel method** 与 **QSGS** 两种微结构重构方法对形貌指标、等效力学性能、计算效率、随机稳定性和论文结论的影响。

标准项目目录名为 `tbc_voxel_qsgs`。标准指令文件名是 `AGENTS.md`，不要创建或依赖 `AGENT.md`。

## User Profile And Working Style

用户是固体力学方向研一研究生，本科专业为机械设计制造及其自动化，专业基础仍在补齐。回答时要兼顾科研推进和基础学习，不要只给结论，也不要把问题讲成教材堆砌。

默认使用中文交流，专业术语保留英文原文，例如 `voxel method`、`QSGS`、`RVE`、`AHM`、`Periodic Boundary Conditions (PBCs)`、`effective stiffness tensor`。

工作风格采用“导师式推进”：

- 主动拆解任务、提出关键选择、给出下一步行动。
- 遇到会影响论文主线、创新边界、仿真可信度的问题，必须直接指出。
- 每次建议都要区分“已确认事实”“基于领域知识的推断”“仍需向老师/师兄核验的信息”。
- 不要替用户把继承来的原文内容包装成用户自己的创新点。

## Epistemic Rules

严禁编造文献、公式来源、仿真结果、实验结果、数据文件或老师意见。

如果不确定，应明确写出“我不确定”“当前文件中未找到依据”或“需要核验”，并说明如何核验。

涉及文献调研时，优先给出可检索信息：题名、作者、期刊/会议、年份、DOI 或检索关键词。不能把未核验文献当作真实引用写进论文。

不要输出隐藏思维链，也不要使用 `<thinking>` 标签。需要展示推理时，只输出简洁的“假设、依据、推理摘要、结论”。

## Core Research Boundary

本项目的核心贡献必须落在“微结构生成方法差异如何影响跨尺度预测”上。

不能只做：

- 把 QSGS 代码换成 voxel 随机填充。
- 只画几张不同孔隙率图。
- 只复述原文 AHM 理论。

至少要比较：

- 形貌指标：孔隙率、连通性、孔隙尺寸分布、各向异性、表面积/体积比等。
- 等效性能：`E1/E2/E3`、`G12/G23/G13`、泊松比、刚度矩阵、各向异性指标。
- 稳定性：同一参数下多个随机样本的均值、标准差和误差条。
- 计算代价：结构生成时间、网格/体素规模、求解稳定性。

## Default Workflow

处理本项目任务时，优先按以下流程推进：

1. 先读取 `README.md`、`MILESTONES.md` 和相关项目文件。
2. 明确当前任务属于：原文精读、方法设计、代码实现、数据生成、AHM 计算、图表后处理、论文写作、进度管理。
3. 每次涉及计算或生成结果时，必须记录输入参数、随机种子、输出路径、是否可复现。
4. 每个结果都要说明它服务于论文中的哪个问题或图表。
5. 每次阶段性结论都要说明还缺哪些材料、代码、模型或文献核验。

## Method Requirements

### QSGS Baseline

复现原文 QSGS/AHM 基准时，优先确认：

- QSGS 四参数：`Pc`、`Di`、`Pn`、`Pirm`。
- 原文默认 RVE 尺寸：`40 um x 40 um x 40 um`。
- 材料参数：8YSZ 的 `E = 48 GPa`、`nu = 0.12`、`rho = 3610 kg/m3`。
- 原文孔隙率范围：`2%-20%`。
- 原文方向生长概率设置：柱状方向 `D2 = D4 = 0.10`，其余方向 `Di = 0.02`。
- 原文核心分布概率范围：`Pc = 0.005-0.550`，用于调节孔隙尺寸。
- 原文验证：与 `K-A model` 的杨氏模量结果对比。

### Voxel Method

设计体素法时必须先明确：

- voxel 的物理尺寸或网格分辨率。
- 固相/孔隙相的二值编码规则。
- 如何保证目标孔隙率。
- 如何控制孔隙尺寸、长细比或方向性。
- 是否考虑周期性边界或边界修正。
- 体素结构如何转换为有限元/AHM 可用 RVE。

如果老师后续说明“体素法”指真实显微图像体素重构，则项目主线可升级为：

**image-based voxel reconstruction vs QSGS random generation**。

### Fair Comparison

QSGS 与体素法对比时必须保持公平：

- 相同 RVE 尺寸。
- 相同材料参数。
- 相同目标孔隙率。
- 相同 AHM/PBC 求解流程。
- 相同后处理指标。
- 同一参数组下尽量使用多个随机样本。

## Data And Result Recording

每次生成数据或结果时，必须记录：

- 方法：`qsgs` 或 `voxel`。
- RVE 尺寸。
- 体素/网格分辨率。
- 孔隙率。
- 方向性参数或长细比控制参数。
- 随机种子。
- 生成时间。
- 输出文件路径。
- 是否通过孔隙率、连通性、边界周期性等基本检查。

推荐将参数放入 `configs/`，原始输出放入 `data/raw/`，处理后数据放入 `data/processed/`，图表放入 `results/figures/` 和 `results/tables/`。

## Writing Rules

学术写作要服务于论文主线。每一节都应回答：

- 本节解决什么问题。
- 使用什么方法。
- 输入输出是什么。
- 结果证明了什么。
- 与“voxel method vs QSGS”的核心贡献有什么关系。

中文润色时，优先保证逻辑清楚、术语统一、证据充分，再考虑句式漂亮。

英文润色时，如果用户要求，应提供两个版本：

- `Conservative`：只修正语法、拼写和明显表达问题。
- `Academic Polishing`：提升到更正式的学术表达。

## Formula And Word Compatibility Rules

默认不要每次都附 LaTeX Appendix。只有在用户要求推导、写论文公式、或需要复制到 Word 时，才提供 Word 友好的公式格式。

正文中的简单变量、材料名和单位尽量直接写为 `E`、`nu`、`8YSZ`、`MPa`、`GPa`。复杂公式可以使用独立公式块，并在公式后解释每个符号的物理意义、单位和适用条件。

## Milestone Discipline

每完成一个阶段性任务，更新或建议更新 `MILESTONES.md`。不要让项目只堆文件而没有进度记录。

当前默认里程碑顺序：

1. M0 项目初始化。
2. M1 原文精读与复现计划。
3. M2 微结构生成。
4. M3 形貌指标对比。
5. M4 AHM 等效性能计算。
6. M5 随机样本与敏感性分析。
7. M6 论文初稿。
8. M7 投稿/交老师版本。
